# Network Mapper

Веб-приложение на Python для сканирования локальной сети и наглядного отображения найденных устройств.

После запуска приложение:

- находит активные локальные подсети текущего компьютера
- сканирует устройства в этих подсетях
- показывает:
  - IP-адрес
  - MAC-адрес
  - имя хоста
  - производителя
  - предположительный тип устройства
- строит интерактивную карту сети
- позволяет задавать понятные пользовательские имена устройствам через mapping-файл `device_aliases.json`

---

## Возможности

- сканирование локальной сети через `nmap`
- отображение прогресса сканирования в реальном времени
- интерактивная карта сети
- подробная таблица устройств
- сохранение пользовательских имён устройств по MAC-адресам
- повторное использование mapping-файла при следующих запусках

---

## Что требуется

На компьютере должны быть установлены:

1. **Windows 10 / Windows 11**
2. **Python 3.10+**
3. **Nmap**
4. Python-пакеты:
   - `flask`
   - `psutil`
   - `python-nmap`

# Запуск на новой машине с нуля

## 1. Установить Python

Скачайте и установите Python.

Во время установки обязательно включите галочку:

- `Add Python to PATH`

Проверка:

```bat
python --version
````

или:

```bat
py --version
```

---

## 2. Установить Nmap

Скачайте и установите `Nmap` для Windows.

После установки проверьте:

```bat
nmap --version
```

Если команда не найдена, нужно добавить папку с `nmap.exe` в `PATH`.

Обычно это одна из папок:

```text
C:\Program Files\Nmap
```

или

```text
C:\Program Files (x86)\Nmap
```

После добавления в `PATH` заново откройте терминал и снова выполните:

```bat
nmap --version
```

---

## 3. Скачать проект

Через Git:

```bat
git clone https://github.com/chiginskiy/Network-mapper.git
cd Network-mapper
```

Если Git не установлен, можно просто скачать ZIP-архив репозитория с GitHub и распаковать его.

---

## 4. Установить зависимости Python

В папке проекта выполните:

```bat
pip install flask psutil python-nmap
```

Если `pip` не срабатывает:

```bat
py -m pip install flask psutil python-nmap
```

---

## 5. Проверить имя основного файла

В папке проекта должен быть основной Python-файл приложения.

Например:

```text
app.py
```

Если файл называется иначе, в командах ниже подставьте его имя.

---

## 6. Запустить PowerShell или cmd от имени администратора

Это важно.

Без прав администратора:

* часть MAC-адресов может не определяться
* `nmap` может работать неполно
* некоторые устройства будут определяться хуже

Как открыть:

1. нажмите **Пуск**
2. введите `PowerShell` или `cmd`
3. нажмите правой кнопкой мыши
4. выберите **Запуск от имени администратора**

---

## 7. Перейти в папку проекта

Например:

```bat
cd C:\Users\Ivan\Downloads\Network-mapper
```

Подставьте свой путь.

---

## 8. Запустить приложение

Если основной файл называется `app.py`:

```bat
python app.py
```

или:

```bat
py app.py
```

После запуска в консоли должно появиться сообщение вида:

```text
Откройте в браузере: http://127.0.0.1:5000
```

---

## 9. Открыть веб-интерфейс

Откройте в браузере:

```text
http://127.0.0.1:5000
```

Нажмите кнопку:

```text
СКАНИРОВАТЬ СЕТЬ
```

Приложение автоматически определит активную локальную сеть текущего компьютера и начнёт сканирование.

---

# Как использовать в другой локальной сети

Ничего менять в коде не нужно.

Приложение само:

* определяет активные сетевые интерфейсы текущей машины
* находит доступные локальные подсети
* сканирует именно ту сеть, к которой подключён этот компьютер

То есть проект можно перенести на другой ПК в другой сети и просто запустить там по той же инструкции.

---

# Файл пользовательских названий устройств

Приложение поддерживает файл:

```text
device_aliases.json
```

Если он лежит рядом с Python-файлом, то из него автоматически подгружаются:

* пользовательские имена устройств
* комментарии к устройствам

Пример:

```json
{
  "version": 1,
  "updated_at": "2026-03-17T12:00:00Z",
  "devices": {
    "74:DA:88:A9:E3:A4": {
      "name": "TP-Link",
      "comment": "Роутер"
    },
    "24:D0:DF:48:4E:D8": {
      "name": "iPhone SE",
      "comment": "Дмитрий"
    }
  }
}
```

### Как с ним работать

1. запустите сканирование
2. заполните имена устройств в таблице
3. нажмите:

   * **Скопировать текущий mapping**
   * или **Сохранить mapping рядом**
4. при следующем запуске эти имена автоматически применятся снова

---

# Если MAC-адреса определяются не у всех устройств

Проверьте:

1. терминал запущен **от имени администратора**
2. `nmap` установлен и доступен в командной строке
3. компьютер подключён к обычной локальной сети
4. антивирус или firewall не блокируют сканирование
5. устройство действительно доступно в той же подсети

---

# Если не работает команда `python`

Попробуйте:

```bat
py app.py
```

Если не работает и это, значит Python не добавлен в `PATH` или установлен некорректно.

---

# Если не работает команда `nmap`

Проверьте:

```bat
nmap --version
```

Если команда не находится:

* переустановите Nmap
* или вручную добавьте папку с `nmap.exe` в `PATH`

После этого обязательно откройте новый терминал.

---

# Быстрый запуск в двух командах

После установки Python и Nmap:

```bat
py -m pip install flask psutil python-nmap
py app.py
```

---

# Примечания

* на Windows лучше запускать от имени администратора
* в больших сетях сканирование может занимать время
* типы устройств определяются эвристически
* если точный шлюз определить не удалось, приложение может использовать предположение `network + 1`

---

# Для чего это полезно

Проект подходит для:

* инвентаризации домашней или офисной сети
* поиска неизвестных устройств
* составления списка устройств по MAC-адресам
* диагностики локальной сети
* визуального понимания, кто находится в сети
